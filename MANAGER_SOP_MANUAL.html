1. INTRODUCTION

1.1 Purpose of This Manual

This Standard Operating Procedure (SOP) manual provides comprehensive guidance for Managers using the Berjaya WMS (Warehouse Management System). It covers all functions, features, and workflows available in the Manager Dashboard.

1.2 Scope

This manual covers:

  • All Manager Dashboard functions and features
  • Step-by-step procedures for common tasks
  • Best practices for daily operations
  • Troubleshooting common issues

1.3 Target Users

User Role Access Level
Managers Full access to all documented features
Supervisors Limited access based on permissions
DevAdmins Full system access including HR module

1.4 System Requirements

  • Modern web browser (Chrome, Firefox, Edge - latest versions)
  • Stable internet connection
  • Valid login credentials with Manager role

2. GETTING STARTED

2.1 Logging In

Procedure: Login to System

  1. Open your web browser
  2. Navigate to the Berjaya WMS URL
  3. Click "Sign in with Google"
  4. Select your authorized Google account
  5. Wait for authentication to complete
  6. You will be redirected to the Manager Dashboard
Use only your authorized company Google account. If login fails, contact your DevAdmin for account verification.

4. INVENTORY MANAGEMENT

4.1 Overview

The Inventory category contains 6 sub-tabs for comprehensive inventory control:

Sub-Tab Purpose
Expected Inventory View baseline inventory by batch and location
Checked Items See items counted by workers
Transaction Log Complete audit trail of movements
Waste Inventory Track waste, lost, and defective items
Scanner Manage barcode scanner data
Item Master Catalog management and BOMs

4.2 Expected Inventory Tab

Purpose: View the current inventory levels based on batch allocations.

Procedure: Viewing Inventory

  1. Click Inventory ‚Üí Expected Inventory
  2. Use filters to narrow your search:
    • Search Box: Enter SKU or Item Name
    • Batch Filter: Select specific batch or "All Batches"
    • Location Filter: Select facility location
  3. Review the summary cards (Total SKUs, Total Quantity, Total Locations)

Procedure: Exporting Data

  1. Apply desired filters
  2. Click Export CSV button
  3. File downloads automatically with current date

4.3 Checked Items Tab

Purpose: View items that workers have physically counted during their shift.

Procedure: View Checked Items

  1. Click Inventory ‚Üí Checked Items
  2. Review the list showing:
    • SKU and Item Name
    • Location
    • Amount Counted
    • Counted By (worker name)
    • Timestamp

4.4 Transaction Log Tab

Purpose: Complete audit trail of all inventory movements.

Transaction Types

Type Description
IN Items received into inventory
OUT Items removed from inventory
ADJUSTMENT Manual quantity corrections
TRANSFER Items moved between locations
RECTIFICATION Correction entry from cancelled transaction

Procedure: Cancelling Transactions

  1. Select transactions using checkboxes
  2. Click Cancel & Rectify button
  3. Enter reason for cancellation
  4. Confirm the action
  5. System creates rectification entry automatically
Only COMPLETED transactions can be cancelled. Cancellation creates a rectification entry to maintain the audit trail.

4.5 Waste Inventory Tab

Purpose: Track and approve waste, lost, defective, and unplanned usage reports.

Report Types

Type Icon Description
WASTE üî• Items discarded due to damage
LOST ‚ùì Items missing or unaccounted
DEFECT ‚ö†Ô∏è Items found defective during QA
UNPLANNED_USAGE üìã Items used outside normal operations

Procedure: Approving Reports

  1. Filter by Status: Pending
  2. Click View Details on report
  3. Review all information and photos
  4. Click Approve Report button
  5. System records your name and timestamp

4.6 Item Master Tab

Purpose: Manage the item catalog and Bill of Materials (BOMs).

Sub-tab: Items

Procedure: Creating/Editing Items

  1. Click Add Item or Edit on existing item
  2. Fill in required fields (SKU, Item Name, Category, Unit)
  3. Click Save

Sub-tab: BOMs (Bill of Materials)

Procedure: Managing BOMs

  1. Click Inventory ‚Üí Item Master ‚Üí BOMs
  2. View existing BOMs or click Add BOM
  3. Define components and quantities
  4. Click Save

5. PRODUCTION LINE MONITORING

5.1 Overview

The Production Line category provides real-time monitoring of manufacturing operations with auto-refresh every 10 seconds.

Sub-Tab Purpose
Takt Time Monitor zone performance and efficiency
Car Tracking Track individual vehicle movements
Starve/Block Analysis Analyze production bottlenecks
Break Schedule Configure operating hours

5.2 Takt Time Tab

Purpose: Monitor production line flow with zone performance metrics.

Zone Status Legend

Status Color Meaning
WORK Green Zone is actively processing
STARVE Yellow Zone waiting for work (no cars)
BLOCK Red Zone blocked by downstream
IDLE Gray Zone not in operation
PAUSED Blue Zone paused for break

Procedure: Monitoring Production

  1. Click Production Line ‚Üí Takt Time
  2. View real-time zone status grid
  3. Monitor key metrics:
    • Expected vs Actual Takt Time
    • Average Processing Time per Zone
    • Line Efficiency Percentage
    • Cars Processed Today
Data auto-refreshes every 10 seconds. Click Refresh for immediate update.

5.3 Car Tracking Tab

Purpose: Track individual vehicle movements through production zones.

Available Views

View Description
Raw Transactions Complete list of all check-in/check-out events
By Car (Search VIN) Full journey timeline for specific vehicle
By Zone All cars currently in each zone
Edit Cars Modify or delete car records

Procedure: Search by VIN

  1. Click By Car - Search VIN view
  2. Enter VIN in search box
  3. View full journey timeline including zone progression, time in each zone, and worker assignments

5.4 Break Schedule Configuration Tab

Purpose: Configure factory operating hours and break schedules.

Setting Description
Factory Hours Start and end time of operations
Morning Break Morning rest period
Lunch Break Midday break period
Afternoon Break Afternoon rest period
Break schedules affect all Takt Time calculations. Ensure accuracy when configuring.

10. COMMON WORKFLOWS

10.1 Daily Inventory Reconciliation

Purpose: Verify inventory accuracy at end of shift.

Workflow Steps

  1. Review Expected Inventory
    • Go to Inventory ‚Üí Expected Inventory
    • Note total quantities by location
  2. Check Worker Counts
    • Go to Inventory ‚Üí Checked Items
    • Compare with expected values
  3. Review Transactions
    • Go to Inventory ‚Üí Transaction Log
    • Filter by today's date
    • Review all movements
  4. Handle Discrepancies
    • Create adjustment transactions if needed
    • Use Cancel & Rectify for incorrect entries
  5. Export Records
    • Export final inventory count to CSV
    • Save for daily records

10.2 Production Monitoring (Continuous)

Purpose: Monitor real-time production status.

Workflow Steps

  1. Check Zone Status - Go to Production Line ‚Üí Takt Time
  2. Identify Issues - Look for STARVE or BLOCK conditions
  3. Track Specific Cars - Go to Car Tracking, search VINs if needed
  4. Review Trends - Check Starve/Block Analysis for bottlenecks
  5. Verify Schedules - Confirm Break Schedule is accurate

10.3 Waste Report Approval Process

Purpose: Review and approve waste/defect reports.

Workflow Steps

  1. Filter Pending Reports - Go to Inventory ‚Üí Waste Inventory, filter by Status: Pending
  2. Review Each Report - Click View Details, check item info and photos
  3. Approve or Investigate - If valid, click Approve Report
  4. Export for Records - Export approved reports to CSV

10.4 New Item Setup

Purpose: Add new items to the system catalog.

Workflow Steps

  1. Access Item Master - Go to Inventory ‚Üí Item Master ‚Üí Items
  2. Create Item - Click Add Item, enter SKU, Name, Category, Unit
  3. Save Item - Click Save, verify item appears in list
  4. Create BOM (if needed) - Go to BOMs tab, add components

11. TROUBLESHOOTING

11.1 Common Issues and Solutions

Issue Solution
Data not appearing Check filters; try different date range or location
Slow loading Reduce items per page; apply stricter filters
Production data stale Click Refresh or wait 10 seconds for auto-update
Cannot approve report Verify manager role; check if already approved
Transaction won't cancel Only COMPLETED non-rectification transactions can be cancelled
Export is empty Check filters - may have excluded all items
Login failed Verify using authorized Google account; contact DevAdmin

11.2 Data Refresh

  • Production data: Auto-refreshes every 10 seconds
  • Inventory data: Real-time sync with Firestore
  • Manual refresh: Click Refresh button on any page

11.3 Browser Issues

If experiencing display issues:

  1. Clear browser cache
  2. Refresh the page
  3. Try different browser
  4. Check internet connection
